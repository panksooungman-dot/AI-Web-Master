# /endday — AI Business OS 프로젝트 종료

`endday`는 **Claude Code 전용** 프로젝트 종료 명령이다.

**Git 작업(add/commit/push)은 절대 수행하지 않는다.** Git은 PowerShell `exit` 명령(`scripts/ai-business-os.ps1`)이 전담한다.
이 명령은 오늘 작업 내용을 분석해 `docs/09_WORK_HISTORY/` 문서를 정리하는 역할만 수행한다.

## 실행 순서

1. **CURRENT_CONTEXT 읽기**
   - `docs/09_WORK_HISTORY/CURRENT_CONTEXT.md`를 읽어 이전 상태(진행 중 작업 · 다음 작업 · 결정사항)를 파악한다.

2. **오늘 Git 변경사항 확인 (읽기 전용)**
   - `git status --porcelain`, `git diff --stat`, `git log --since="today 00:00" --oneline`으로 오늘 변경된 파일과 커밋을 확인한다.
   - 이 단계에서 `add`/`commit`/`push`/`checkout` 등 저장소 상태를 바꾸는 명령은 절대 실행하지 않는다.

3. **오늘 작업 내용 분석**
   - 위에서 확인한 변경 파일 + 이번 세션 대화에서 실제로 완료한 작업을 근거로 오늘 완료한 작업을 정리한다.
   - 실제로 하지 않은 작업이나 확인되지 않은 수치를 추측해서 기록하지 않는다.

4. **내일 작업(TODO) 생성**
   - 아래 우선순위로 정리한다.
     1. 현재 진행 중인 작업 (오늘 마무리하지 못하고 이어지는 것)
     2. 미완료 작업 (계획했으나 오늘 시작하지 못한 것)
     3. 새 기능 (오늘 작업 중 새로 드러난 후속 작업·개선 아이디어)
   - 이 TODO를 5·6·7단계 문서에 동일하게 반영한다.

5. **`docs/09_WORK_HISTORY/CURRENT_CONTEXT.md` 업데이트 — 항상 최신 상태만 유지 (덮어쓰기)**
   - 이 문서는 이력이 아니라 "지금 상태 스냅샷"이다. 과거 내용을 아래에 쌓지 않고, 아래 섹션들을 오늘 기준 최신 내용으로 **교체**한다.
     - `현재 상태 > 완료`: 오늘까지 완료된 것 중 여전히 유효한 항목만 남기고 최신화(오래되어 무의미해진 완료 항목은 정리 가능)
     - `현재 진행 중`: 지금 시점에 실제로 진행 중인 것만
     - `다음 작업`: 4번에서 만든 TODO로 전체 교체
     - `AI 인수인계`: 다음 세션이 이어받을 핵심 맥락으로 갱신
     - `Last Update`: 오늘 날짜로 갱신
   - 오래된 정보를 남겨두지 않는다 — 다음 세션은 이 문서만 보고 바로 이어갈 수 있어야 한다.

6. **`docs/09_WORK_HISTORY/WORK_HISTORY.md` 업데이트 — 날짜별 Append, 기존 기록 삭제 금지**
   - "작업 기록" 아래에 오늘 날짜(`## YYYY-MM-DD`) 섹션을 **추가**한다.
   - 오늘 날짜 섹션이 이미 있으면(같은 날 여러 번 실행) 그 섹션만 갱신하고, 다른 날짜 섹션은 절대 수정·삭제하지 않는다.
   - 완료한 작업 / 수정한 주요 파일 / 주요 결정사항 / 해결한 문제 / 다음 작업 / 진행률을 기록한다.
   - "마지막 업데이트"를 오늘 날짜로 갱신한다.

7. **세션 파일 생성/업데이트**
   - `docs/09_WORK_HISTORY/sessions/MM-DD.md`(오늘 날짜)가 있으면 업데이트하고, 없으면 기존 세션 형식(`sessions/2026-07-06.md` 참고)에 맞춰 새로 생성한다.
   - 오늘 목표 / 완료한 작업 / 수정한 파일 / 해결한 문제 / 오늘 결정한 사항 / 현재 상태 / 다음 작업 / 작업 시간 / AI 인수인계 / Last Update 포함.

8. **프로젝트 Health Check 실행 (읽기 전용)**
   - 다음 항목을 확인한다.
     - CURRENT_CONTEXT.md의 Last Update가 오늘 날짜인지 (최신 여부)
     - WORK_HISTORY.md에 오늘 날짜 섹션이 실제로 추가됐는지 (업데이트 여부)
     - `sessions/MM-DD.md`(오늘)가 실제로 존재하는지 (저장 여부)
     - `docs/01_PMO/WBS.md`가 오늘 작업 내용과 어긋나지 않는지 (최신 여부 — 어긋나면 갱신 필요 항목으로 안내만 하고 직접 수정은 하지 않음)
     - Git 커밋되지 않은 변경사항 유무 / 원격 대비 ahead 커밋 수 (Push 여부, **참고용** — 이 값으로 git 조작을 하지 않는다)
   - 위 항목 중 문제가 있으면 완료 메시지 앞에 경고로 안내한다.

9. **완료 메시지 출력**
   - 아래 형식 그대로 출력한다.

```
====================================
AI Business OS - End Day
====================================

✓ CURRENT_CONTEXT 업데이트
✓ WORK_HISTORY 업데이트
✓ Session 저장
✓ Tomorrow TODO 생성
✓ Project Health Check 완료

오늘 작업이 정상적으로 저장되었습니다.
좋은 하루 보내세요.
====================================
```

## 제약 사항

- `git add` / `git commit` / `git push` / `git checkout` 등 저장소 상태를 변경하는 Git 명령은 실행하지 않는다. (Git은 `exit`가 담당)
- 문서 갱신 외의 코드 변경(리팩터링 등)은 수행하지 않는다.
- 사실이 아닌 내용(하지 않은 작업, 임의의 수치)을 문서에 기록하지 않는다.
- CURRENT_CONTEXT.md는 덮어쓰기(최신 상태만), WORK_HISTORY.md는 append(기존 기록 보존) — 이 둘의 갱신 방식을 서로 바꿔 적용하지 않는다.
- 완료 후 대기한다. 추가 작업은 사용자가 요청할 때만 수행한다. (CLAUDE.md Stop Condition)
