# Claude Code 프로젝트 운영 규칙 (AGENTS.md)

> 이 파일은 Claude Code가 프로젝트를 운영하는 표준 규칙이다.  
> 세션 시작 시 반드시 읽고 따른다.

---

## 1. 세션 시작 시 필수 수행

프로젝트를 시작하거나 재개할 때 아래 순서대로 문서를 읽는다.

```
1. AGENTS.md   ← 현재 문서 (운영 규칙)
2. WBS.md      ← 현재 작업 단계 및 진행률 확인
3. REQUEST.md  ← 의뢰자 요구사항 확인 (최초 1회 또는 변경 시)
```

> WBS.md의 **"현재 작업"** 섹션을 가장 먼저 확인한다.  
> 이전에 어디까지 완료했는지 파악한 뒤 작업을 시작한다.

---

## 2. 핵심 운영 규칙

### 2.1 작업 범위 규칙

| 규칙 | 설명 |
|------|------|
| **WBS 기준 개발** | 반드시 WBS.md에 정의된 작업 단위 기준으로 개발한다 |
| **완료 작업 재개발 금지** | 이미 완료(✅)된 항목은 다시 개발하지 않는다 |
| **현재 작업만 수행** | WBS의 현재 작업 항목만 수행한다. 다음 단계를 임의로 선행하지 않는다 |
| **요청 외 리팩터링 금지** | 사용자가 요청하지 않은 리팩터링·최적화·구조 변경을 하지 않는다 |

### 2.2 파일 접근 규칙

| 규칙 | 설명 |
|------|------|
| **필요한 파일만 확인** | 작업에 직접 관련된 파일만 읽는다. 전체 프로젝트를 반복 탐색하지 않는다 |
| **반복 분석 금지** | 같은 파일을 불필요하게 반복해서 읽지 않는다 |
| **토큰 최소화** | 파일 읽기·쓰기는 목적이 명확할 때만 수행한다 |

### 2.3 승인 필요 작업

아래 작업은 반드시 사용자 승인을 받은 뒤 진행한다.

- `package.json` 수정
- 새로운 라이브러리/패키지 설치
- 환경 변수(`.env.local`) 추가·변경
- 데이터베이스 스키마 변경
- 배포 설정 변경

### 2.4 작업 완료 후 필수 수행

```
1. WBS.md 업데이트 — 완료 항목에 ✅ 체크, 다음 작업 항목 업데이트
2. 진행률(%) 업데이트
3. 작업 보고 (아래 형식)
```

## 4. 개발 원칙

### 4.1 컴포넌트 작성

- 모든 컴포넌트는 기본적으로 **Server Component**로 작성한다
- `useState`, `useEffect`, 이벤트 핸들러가 필요한 경우에만 `"use client"` 선언
- `any` 타입 사용 금지 — `unknown` + 타입 가드를 사용한다

### 4.2 스타일

- **Tailwind CSS** 유틸리티 클래스만 사용한다
- 인라인 스타일(`style={{}}`)은 동적 값이 불가피한 경우에만 사용한다
- 컬러 팔레트는 `CNBIZ_RULES.md` 기준을 따른다

### 4.3 이미지

- `<img>` 태그 금지 — `next/image`의 `<Image>` 컴포넌트를 사용한다
- 장식용 SVG에는 `aria-hidden` 속성을 반드시 추가한다

### 4.4 Next.js 16 준수

- 코드 작성 전 `node_modules/next/dist/docs/` 관련 가이드를 참조한다
- `params`, `searchParams`는 `Promise<...>` 타입으로 처리한다

## 문서 관리

- 새로운 문서는 반드시 사용자 승인 후 생성한다.
- 기존 문서를 우선 활용하고 불필요한 문서를 만들지 않는다.
- 문서가 중복되지 않도록 관리한다.
- Git Commit, Push는 반드시 사용자 승인 후 진행한다.
- 기존 문서를 삭제하거나 이름을 변경할 경우 반드시 사용자 승인을 받는다.

## 콘텐츠 작성 원칙

- 기업 홈페이지의 문구는 전문 카피라이터 수준으로 작성한다.
- 브랜드 철학과 신뢰를 우선한다.
- 과장된 마케팅 문구보다 명확하고 설득력 있는 표현을 사용한다.
