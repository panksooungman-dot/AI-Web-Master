# Token Policy

AI 에이전트가 컨텍스트·토큰을 효율적으로 사용하기 위한 회사 표준 정책입니다.

> 루트 `AGENTS.md` §2.2(파일 접근 규칙)의 "토큰 최소화" 원칙을 실행 단위로 구체화한 문서입니다. 해당 원칙은 여기서 반복하지 않고 참조만 합니다.

---

## 1. Purpose

토큰·컨텍스트는 유한한 자원입니다. 불필요한 소비는 응답 속도·비용·품질 저하로 이어집니다. 본 정책은 모든 에이전트(Planner/Builder/Reviewer/Architect/Documenter)에 동일하게 적용됩니다.

---

## 2. File Access Rules

| 규칙 | 설명 |
|------|------|
| 목적 기반 읽기 | 작업에 직접 필요한 파일만 읽는다. 탐색을 위한 전체 순회를 지양한다 |
| 검색 우선 | 파일 전체를 읽기 전에 검색(키워드·패턴)으로 관련 위치를 먼저 좁힌다 |
| 부분 읽기 | 큰 파일은 관련 라인 범위만 읽는다. 전체를 읽어야 할 명확한 이유가 없으면 부분만 읽는다 |
| 재읽기 금지 | 같은 세션에서 변경하지 않은 파일을 다시 읽지 않는다 |
| 출력 재확인 금지 | 직접 쓰기/수정한 파일은 결과를 다시 읽어 검증하지 않는다 (실패 시 도구가 오류를 반환함) |

---

## 3. Context Window Management

- 대화가 길어져 컨텍스트 한계에 가까워지는 것은 세션 관리 계층이 자동 처리한다. 에이전트가 선제적으로 대화를 요약·축약하지 않는다.
- 이미 확립된 사실(이전 턴에서 확인한 내용)은 다시 조사하지 않고 그대로 활용한다.
- 같은 결정을 재논의하지 않는다 — 이미 승인된 방향은 재확인 없이 진행한다.

---

## 4. Delegation Policy (서브 에이전트 위임 기준)

| 상황 | 위임 여부 |
|------|-----------|
| 결과만 필요하고 중간 탐색 과정은 메인 컨텍스트에 남길 필요가 없는 조사 | 위임 (fork 또는 조사 전용 에이전트) |
| 여러 독립적 질문을 동시에 처리해야 하는 경우 | 병렬 위임 |
| 간단한 단일 파일 조회·수정 | 위임하지 않고 직접 처리 |
| 위임한 작업의 중간 산출물(로그·트랜스크립트)은 | 직접 열람하지 않고 완료 보고만 신뢰한다 |

세부 오케스트레이션 시점은 `WORKFLOW.md`를 따른다.

---

## 5. Output Verbosity Policy

- 파일 내용 전체를 그대로 다시 출력하지 않는다. 변경 요약으로 대체한다.
- 사용자가 요청하지 않은 부가 설명·재요약을 덧붙이지 않는다.
- 표·다이어그램은 정보 밀도가 텍스트보다 높을 때만 사용한다.

---

## 6. Prohibited Patterns

- 동일 파일을 근거 없이 반복 조회
- 요청 범위를 벗어난 전체 저장소 탐색
- 이미 확인된 내용을 "검증"이라는 명목으로 재조회
- 결론 없이 과정만 장황하게 보고
