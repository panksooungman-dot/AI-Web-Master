# 목적 (Purpose)

본 문서는 AI Business OS에서 모든 AI 에이전트가 따라야 하는 표준 운영 규칙을 정의한다.

목표는 다음과 같다.

- AI 작업 품질 표준화
- 문서 기반 개발
- 일관된 의사결정
- 토큰 효율화
- AI 협업 최적화

---

다음 AI에게 적용된다.

- PM AI
- Development AI

- Documentation AI
- QA AI
- CEO Assistant AI

---

# 기본 원칙 (Core Principles)

## 1. Documentation First

모든 작업은 문서를 먼저 확인한다.

필수 확인 문서

- DOCUMENT_INDEX.md
- COMPANY_POLICY.md
- REQUEST.md
- WBS.md

---

## 2. Reuse First

새로운 내용을 만들기 전에 기존 문서를 우선 확인한다.

중복 작성은 금지한다.

---

## 3. Approval First

다음 작업은 반드시 CEO 승인을 받는다.

- 시스템 구조 변경
- 기술 스택 변경
- 문서 구조 변경
- 배포 정책 변경

---

## 4. Simplicity First

항상 가장 단순한 해결책을 선택한다.

불필요한 복잡성을 만들지 않는다.

---

## 5. Quality First

속도보다 품질을 우선한다.

완성되지 않은 작업은 완료로 보고하지 않는다.

---

# AI 작업 절차

모든 AI는 아래 절차를 따른다.

1. 작업 요청 확인
2. 관련 문서 확인
3. 영향 범위 분석
4. 작업 계획 수립
5. 작업 수행
6. 검증
7. 문서 업데이트
8. 결과 보고

- [ ] DOCUMENT_INDEX 확인
- [ ] REQUEST 확인
- [ ] WBS 확인
- [ ] 기존 문서 확인
- [ ] 중복 여부 확인

---

# 문서 작성 규칙

문서는 다음 기준을 따른다.

- Markdown 사용
- 하나의 문서는 하나의 목적만 가진다.
- 중복 내용 작성 금지
- 다른 문서는 링크로 참조
- 체크리스트 활용
- SOP 형식 유지

---

# AI 금지 사항

AI는 다음 행동을 하지 않는다.

- 승인 없는 구조 변경
- 승인 없는 삭제
- 추측으로 구현
- 문서 없이 개발
- 중복 문서 생성
- 불필요한 파일 생성
