# Context7 MCP Integration

## Overview

The Context7 MCP server enables AI Business OS to retrieve the latest official documentation, SDK references, framework guides, and API specifications for modern software development.

It provides AI Agents with version-aware, authoritative technical knowledge to improve implementation accuracy and reduce outdated recommendations.

---

# Purpose

Provide standardized access to current technical documentation for libraries, frameworks, programming languages, and cloud platforms.

---

# Objectives

- Retrieve official documentation
- Reduce outdated information
- Improve implementation accuracy
- Support version-aware development
- Standardize technical references
- Accelerate developer productivity

# Documentation Priority

Always consult documentation in the following order:

1. Official documentation
2. Official SDK/API reference
3. Official migration guides
4. Official release notes
5. Trusted ecosystem documentation

Avoid relying on unofficial or outdated sources when official documentation is available.

---

# Version Policy

Always:

- Identify the target version.
- Use version-specific documentation.
- Verify API compatibility.
- Review migration notes before upgrades.

Never:

- Mix documentation from incompatible versions.
- Assume backward compatibility.
- Ignore deprecation warnings.

---

# Documentation Retrieval Guidelines

When retrieving documentation:

- Prefer official sources.
- Retrieve only relevant sections.
- Include implementation examples.
- Highlight breaking changes.
- Note deprecated APIs.

---

# Code Example Policy

Examples should:

- Follow official recommendations.
- Be minimal and reproducible.
- Match the target version.
- Include required imports.
- Demonstrate best practices.

Avoid:

- Deprecated APIs
- Experimental features unless requested
- Incomplete snippets
- Unverified examples

---

# Validation Checklist

Before applying documentation:

- Technology identified
- Version confirmed
- Documentation is official
- APIs are current
- Examples are compatible
- Breaking changes reviewed

---

# Error Handling

If documentation cannot be retrieved:

1. Verify the technology name.
2. Confirm the target version.
3. Retry retrieval.
4. Use the nearest supported version if appropriate.
5. Record limitations.
6. Notify the responsible Agent.

---

# Best Practices

- Always verify library versions.
- Prefer official examples.
- Check migration guides before upgrades.
- Review release notes regularly.
- Update Knowledge Memory with validated patterns.
- Revalidate documentation after major releases.

---

# Success Criteria

Context7 integration is successful when:

- Official documentation is retrieved.
- Version compatibility is confirmed.
- Code examples are accurate.
- AI recommendations reflect current best practices.
- Technical guidance is traceable and reproducible.
