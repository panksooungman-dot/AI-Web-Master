# Knowledge Memory

## Overview

Knowledge Memory stores reusable knowledge, best practices, lessons learned, reference materials, and organizational expertise used throughout AI Business OS.

Unlike Project Memory, which is project-specific, Knowledge Memory captures general knowledge that can be applied across multiple projects.

---

# Purpose

Preserve long-term organizational knowledge that improves consistency, reduces repeated work, and enables continuous learning.

---

# Knowledge Categories

Knowledge may include:

- Software Engineering
- System Architecture
- AI Engineering
- Business Analysis
- Product Management
- DevOps
- Quality Assurance
- Technical Writing
- Security
- Performance Optimization

# Best Practices

Always:

- Document verified knowledge.
- Prefer reusable guidance.
- Keep examples practical.
- Record lessons learned.
- Organize knowledge by category.
- Review periodically.

# Update Rules

Update Knowledge Memory whenever:

- A new best practice is identified.
- A reusable solution is created.
- A lesson is learned.
- An architectural pattern is validated.
- New technologies are adopted.
- Existing guidance becomes outdated.

---

# Retrieval Rules

When retrieving knowledge:

1. Prefer validated knowledge.
2. Use the latest approved guidance.
3. Reuse existing patterns before creating new ones.
4. Cite related documentation.
5. Keep recommendations technology-appropriate.

---

# Quality Guidelines

Every knowledge entry should be:

- Accurate
- Practical
- Reusable
- Well organized
- Easy to understand
- Version controlled

---

# Constraints

Never:

- Store unverified information.
- Duplicate Project Memory.
- Preserve obsolete guidance without marking it.
- Contradict approved standards.
- Mix project-specific information with general knowledge.
