```
  site-config.ts         # Site identity, nav links, legal links
styles/
  tokens.ts               # Raw design tokens (colors, typography, spacing, radius)
  theme.ts                 # Semantic theme combining tokens + site identity
  theme.css                 # Tailwind v4 @theme mapping (source of truth for colors)
public/               # Static assets — logo.svg, manifest.json, placeholder images
```

## Design System

Colors, typography, spacing, and radius live in `styles/tokens.ts` and are mirrored
as CSS custom properties in `styles/theme.css` (imported by `app/globals.css`), which
feeds Tailwind's `@theme` block — so `bg-primary`, `text-foreground`, `border-border`,
etc. are available as utility classes everywhere. `styles/theme.ts` re-exports the
tokens alongside site identity (name, type) as a single `theme` object.

## Content

All page copy lives in `lib/content.ts`, generated once by the Website Builder's
Content Engine (Provider Layer) at `ai website create` time. Edit it directly to
change any headline, feature, testimonial, plan, FAQ, or blog excerpt — every page
and component reads from this one file, so there's a single source of truth.

## SEO

- Per-page `metadata` (title/description) on every route
- `app/robots.ts` → `/robots.txt`
- `app/sitemap.ts` → `/sitemap.xml`
- `app/opengraph-image.tsx` → dynamic social share image
- `public/manifest.json` → PWA manifest
- Organization JSON-LD (`components/JsonLd.tsx`, rendered in the root layout)

## Deployment

1. Copy `.env.example` to `.env.local` (or configure the same variables in your
   hosting provider) and set `NEXT_PUBLIC_SITE_URL` to your real domain.
2. `npm run build` to verify a production build locally.
3. Deploy — `vercel.json` is preconfigured for zero-config deployment to
   [Vercel](https://vercel.com), or run `npm run build && npm run start` on any
   Node.js host.
