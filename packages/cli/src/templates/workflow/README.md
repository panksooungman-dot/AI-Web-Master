# {{className}} Workflow

Generated by AI Business OS CLI (`ai create workflow {{name}}`).

Run it with:

```
ai workflow run {{name}}
```

## workflow.json

Steps run **sequentially**, top to bottom. Each step names an Agent by its
`agents/<name>` (or `marketplace/agents/<name>`) package:

```json
{
  "name": "{{name}}",
  "version": "{{version}}",
  "steps": [
    { "agent": "business-analyst" },
    { "agent": "ui-designer" },
    { "agent": "frontend-engineer" }
  ]
}
```

If a step's agent has not been created/installed yet (`ai create agent <name>` /
`ai install <name>`), the Workflow Runtime runs it as a simulated placeholder
step instead of failing — useful for sketching a pipeline before every agent
exists. If an agent package exists but is invalid (missing files, bad
metadata/version), the workflow stops on that step (Stop on error).

## steps/

Place per-step assets here (notes, overrides, fixtures) as your workflow
grows. The engine does not read this folder yet — it is reserved for future
per-step customization.
