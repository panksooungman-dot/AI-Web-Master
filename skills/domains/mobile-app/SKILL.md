# Mobile App Domain

## Purpose

This skill defines best practices for designing, developing, testing, deploying, and maintaining modern mobile applications.

The objective is to deliver secure, performant, scalable, and user-friendly mobile experiences across iOS and Android platforms.

---

# When to Use

Execute when:

- Building native mobile applications
- Developing cross-platform applications
- Creating companion mobile apps
- Supporting mobile-first products
- Publishing apps to app stores

---

# Objectives

Build mobile applications that are:

- Fast
- Reliable
- Secure
- Accessible
- Maintainable

---

# Inputs

Expected inputs:

- Business Requirements
- UX/UI Designs
- Mobile Platform Requirements
- API Specifications
- Security Requirements
- Performance Targets

---

# Core Capabilities

Support:

- User Authentication
- Offline Mode
- Push Notifications
- Deep Linking
- Background Tasks
- Camera & Media
- Location Services
- Secure Storage

---

# Platform Support

Target:

- iOS
- Android
- Tablet Devices (if applicable)

Maintain platform-specific user experience where appropriate.

---

# User Experience

Ensure:

- Responsive Layouts
- Smooth Navigation
- Fast Startup
- Intuitive Gestures
- Consistent Design
- Accessibility Support

Optimize for mobile interaction patterns.

---

# Performance

Optimize:

- Startup Time
- Rendering Performance
- Battery Usage
- Memory Usage
- Network Requests
- App Size

Measure before optimizing.

---

# Security

Implement:

- Secure Authentication
- Encrypted Storage
- Secure API Communication
- Certificate Validation
- Biometric Authentication
- Sensitive Data Protection

Never store sensitive data insecurely.

---

# Offline Capability

Support:

- Local Data Storage
- Synchronization
- Conflict Resolution
- Retry Mechanisms
- Graceful Offline Experience

Provide continuity when connectivity is unavailable.

---

# Device Integration

Integrate with:

- Camera
- Photo Library
- GPS
- Contacts (if required)
- Notifications
- File System

Request only necessary permissions.

---

# Testing

Perform:

- Unit Testing
- UI Testing
- Device Testing
- Cross-platform Testing
- Performance Testing
- Accessibility Testing

Test across multiple screen sizes and OS versions.

# Deployment

Prepare for:

- App Store Review

- Version Management
- Release Notes
- Crash Monitoring
- Analytics

Follow platform publishing guidelines.

# Validation Checklist

Before completion verify:

- Authentication tested
- Offline mode validated
- Performance targets achieved
- Security reviewed
- Accessibility verified
- Store submission ready

---

# Failure Conditions

Stop and request clarification if:

- Platform requirements are undefined
- API contracts are incomplete
- Performance targets are unknown
- Security requirements are missing
- Store release requirements are incomplete

---

# Rules

- Design mobile-first experiences.
- Optimize battery and network usage.
- Protect user data at all times.
- Test on real devices whenever possible.
- Follow platform-specific design guidelines.

---

# Success Criteria

This skill succeeds when:

- the application performs reliably on supported devices
- user experience is intuitive
- security requirements are satisfied
- app store deployment succeeds
- post-release quality metrics meet expectations
