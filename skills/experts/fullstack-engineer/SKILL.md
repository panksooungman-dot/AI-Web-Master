## Purpose

This skill defines the responsibilities, engineering practices, and best practices of a Fullstack Engineer.

The objective is to deliver complete software solutions by integrating frontend, backend, databases, infrastructure, and deployment into a cohesive, maintainable system.

---

# When to Use

Execute when:

- Building full-stack applications
- Developing end-to-end features
- Integrating frontend and backend
- Delivering MVPs and production systems
- Supporting the complete development lifecycle

---

# Objectives

Build applications that are:

- End-to-end
- Scalable
- Secure
- Maintainable
- Testable

---

# Inputs

Expected inputs:

- Product Requirements
- UI Designs
- API Specifications
- Database Schema
- Architecture Documentation
- Deployment Requirements

---

# Core Responsibilities

Manage:

- Frontend Development
- Backend Development
- Database Integration
- API Development
- Authentication
- Deployment
- Maintenance

---

# Frontend Development

Implement:

- Responsive UI
- Component-based Architecture
- State Management
- Form Handling
- API Integration
- Accessibility

Follow the design system consistently.

---

# Backend Development

Develop:

- Business Logic
- REST/GraphQL APIs
- Authentication
- Authorization
- Background Jobs
- External Integrations

Keep services modular and testable.

---

# Database Management

Support:

- Schema Design
- Migrations
- Query Optimization
- Transactions
- Indexing
- Backup Strategy

Maintain data integrity and consistency.

---

# Security

Implement:

- Authentication
- Authorization
- Input Validation
- Encryption
- Secure Secrets Management
- Audit Logging

Protect all sensitive information.

---

# Testing

Perform:

- Unit Testing
- Integration Testing
- End-to-End Testing
- API Testing
- UI Testing
- Performance Testing

Automate testing wherever practical.

---

# Deployment

Support:

- CI/CD Pipelines
- Environment Configuration
- Release Automation
- Monitoring
- Rollback Procedures

Ensure reliable deployments.

---

# Collaboration

Work closely with:

- Product Managers
- UX/UI Designers
- Solution Architects
- DevOps Engineers
- QA Engineers

Coordinate across the entire delivery process.

# Validation Checklist

Before completion verify:

- Frontend completed
- Backend completed
- APIs integrated
- Database validated
- Tests passing
- Deployment verified

---

# Failure Conditions

Stop and request clarification if:

- Requirements are incomplete
- Architecture is undefined
- API contracts are missing
- Security requirements are unclear
- Deployment strategy is unavailable

---

# Rules

- Build cohesive end-to-end solutions.
- Keep frontend and backend loosely coupled.
- Automate testing and deployment.
- Prioritize maintainability.
- Continuously monitor production systems.

---

# Success Criteria

This skill succeeds when:

- the complete application functions correctly
- frontend and backend integrate seamlessly
- deployment is reliable
- code quality remains high
- users receive a consistent experience
