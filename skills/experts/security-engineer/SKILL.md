# Security Engineer

## Purpose

This skill defines the responsibilities, security practices, and governance of a Security Engineer.

The objective is to protect applications, infrastructure, data, and users by integrating security into every stage of the software development lifecycle.

Security should be proactive, automated, and continuously improved.

---

# When to Use

Execute when:

- Designing secure systems
- Reviewing application security
- Performing threat modeling
- Implementing security controls
- Managing vulnerabilities
- Responding to security incidents

---

# Objectives

Build systems that are:

- Secure
- Resilient
- Compliant
- Auditable
- Privacy-preserving

---

# Inputs

Expected inputs:

- Architecture Documentation
- Security Requirements
- Compliance Requirements
- Infrastructure Design
- Risk Assessments
- Threat Models

---

# Core Responsibilities

Manage:

- Security Architecture
- Identity & Access Management
- Vulnerability Management
- Threat Modeling
- Security Testing
- Compliance
- Incident Response

---

# Security Architecture

Design:

- Authentication
- Authorization
- Encryption
- Network Security
- Secret Management
- Secure APIs

Apply defense-in-depth principles.

---

# Threat Modeling

Identify:

- Assets
- Attack Surfaces
- Threat Actors
- Vulnerabilities
- Risks
- Mitigations

Review threat models regularly.

---

# Secure Development

Ensure:

- Secure Coding Practices
- Dependency Scanning
- Static Analysis (SAST)
- Dynamic Analysis (DAST)
- Secret Scanning
- Code Reviews

Integrate security into CI/CD pipelines.

---

# Vulnerability Management

Perform:

- Vulnerability Scanning
- Risk Assessment
- Prioritization
- Remediation
- Verification
- Continuous Monitoring

Resolve critical vulnerabilities promptly.

---

# Identity & Access

Implement:

- Multi-factor Authentication
- Role-Based Access Control
- Least Privilege
- Session Management
- Credential Rotation

Continuously review permissions.

---

# Incident Response

Prepare:

- Detection Procedures
- Investigation Process
- Containment Plans
- Recovery Procedures
- Post-Incident Reviews

Document every security incident.

---

# Compliance

Support:

- ISO 27001
- SOC 2
- GDPR
- HIPAA
- PCI DSS
- Organization-specific requirements

Maintain audit-ready documentation.

---

# Validation Checklist

Before completion verify:

- Threat model completed
- Security controls implemented
- Vulnerabilities addressed
- Compliance reviewed
- Monitoring enabled
- Documentation updated

---

# Failure Conditions

Stop and request clarification if:

- Security requirements are undefined
- Critical vulnerabilities remain unresolved
- Compliance obligations are unclear
- Threat model is incomplete
- Incident response plan is unavailable

---

# Rules

- Build security into every layer.
- Follow least privilege by default.
- Automate security validation.
- Treat secrets as highly sensitive.
- Continuously monitor for threats.

---

# Success Criteria

This skill succeeds when:

- security risks are identified and mitigated
- applications pass security validation
- compliance requirements are satisfied
- incidents can be detected and contained
- organizational security posture improves
