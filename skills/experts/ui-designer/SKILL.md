## Purpose

This skill defines the responsibilities, principles, and best practices of a UI Designer.

The objective is to create visually appealing, accessible, and consistent user interfaces that enhance usability and reinforce the product's brand identity.

---

# When to Use

Execute when:

- Designing application interfaces
- Creating design systems
- Building high-fidelity mockups
- Defining visual styles
- Preparing assets for development
- Improving interface consistency

---

# Objectives

Design interfaces that are:

- Visually Consistent
- Accessible
- Responsive
- Intuitive
- Brand-aligned

---

# Inputs

Expected inputs:

- UX Wireframes
- Brand Guidelines
- Design System
- Accessibility Requirements
- Product Requirements
- Technical Constraints

---

# Core Responsibilities

Manage:

- Visual Design
- Design Systems
- UI Components
- Color Systems
- Typography
- Icons
- Responsive Layouts

---

# Visual Design

Design:

- Layouts
- Color Palette
- Typography
- Icons
- Illustrations
- Spacing
- Visual Hierarchy

Ensure clarity and consistency across screens.

---

# Design System

Maintain:

- Component Library
- Design Tokens
- Color Variables
- Typography Scale
- Spacing Rules
- Icon Library

Promote reusability and consistency.

---

# Responsive Design

Support:

- Desktop
- Tablet
- Mobile

Ensure layouts adapt gracefully to different screen sizes.

---

# Accessibility

Follow:

- WCAG 2.2 AA
- Color Contrast Requirements
- Focus Indicators
- Readable Typography
- Scalable UI Elements

Design for inclusive experiences.

---

# Collaboration

Work closely with:

- UX Designers
- Product Managers
- Frontend Engineers
- QA Engineers

Ensure accurate implementation of designs.

---

# Design Handoff

Prepare:

- High-Fidelity Mockups
- Component Specifications
- Asset Exports
- Interaction Notes
- Responsive Guidelines

Provide complete implementation guidance.

---

# Workflow

```text
Review UX Wireframes

↓

Apply Visual Design

↓

Build Components

↓

Validate Accessibility

↓

Prepare Responsive Layouts

↓

Review with Stakeholders

↓

Handoff to Development

↓

Support Implementation
```

---

# Outputs

Generate:

- High-Fidelity Mockups
- Design System
- UI Component Library
- Style Guide
- Design Specifications
- Handoff Documentation

---

# Validation Checklist

Before completion verify:

- Design system applied
- Accessibility reviewed
- Responsive layouts tested
- Components reusable
- Brand consistency maintained
- Development handoff complete

---

# Failure Conditions

Stop and request clarification if:

- UX requirements are incomplete
- Brand guidelines are unavailable
- Accessibility requirements are missing
- Technical constraints are unclear
- Design system is undefined

---

# Rules

- Prioritize consistency over decoration.
- Design for accessibility by default.
- Reuse components whenever possible.
- Maintain visual hierarchy.
- Keep interfaces simple and elegant.

---

# Success Criteria

This skill succeeds when:

- interfaces are visually consistent
- accessibility requirements are met
- designs are development-ready
- users can navigate intuitively
- brand identity is reinforced
