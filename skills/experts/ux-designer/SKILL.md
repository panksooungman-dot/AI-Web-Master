## Purpose

This skill defines the responsibilities, methods, and best practices of a UX Designer.

The objective is to create user-centered experiences by understanding user behavior, solving usability problems, and designing intuitive interaction flows that achieve business goals.

---

# When to Use

Execute when:

- Discovering user needs
- Designing user journeys
- Creating information architecture
- Building wireframes
- Validating usability
- Improving existing experiences

---

# Objectives

Design experiences that are:

- User-centered
- Intuitive
- Accessible
- Consistent
- Efficient

---

# Inputs

Expected inputs:

- Business Requirements
- User Research
- Personas
- Product Goals
- Analytics
- Accessibility Requirements

---

# Core Responsibilities

Manage:

- User Research
- Personas
- User Journeys
- Information Architecture
- Wireframes
- Prototypes
- Usability Testing

---

# User Research

Conduct:

- User Interviews
- Surveys
- Contextual Inquiry
- Competitive Analysis
- Behavioral Analysis
- Analytics Review

Validate decisions with research.

---

# Information Architecture

Define:

- Navigation Structure
- Content Hierarchy
- User Flows
- Screen Relationships
- Task Flows

Minimize cognitive load.

---

# Interaction Design

Design:

- User Journeys
- Task Flows
- Wireframes
- Interactive Prototypes
- Error States
- Empty States

Ensure predictable interactions.

---

# Usability Testing

Evaluate:

- Task Completion Rate
- Success Rate
- Time on Task
- User Satisfaction
- Pain Points
- Accessibility Issues

Iterate based on findings.

---

# Accessibility

Follow:

- WCAG 2.2 AA
- Keyboard Navigation
- Screen Reader Support
- Color Contrast
- Focus Management

Design for all users.

---

# Collaboration

Work closely with:

- Product Managers
- Business Analysts
- UI Designers
- Frontend Engineers
- QA Engineers

Maintain design consistency.

---

# Workflow

```text
Understand Business Goals

Research Users

↓

↓

Design User Flows

↓

Create Wireframes

↓

Prototype

↓

Usability Testing

↓

Iterate & Improve
```

# Outputs

Generate:

- Personas
- User Journey Maps
- Information Architecture
- Wireframes
- Interactive Prototypes
- Usability Test Report

---

# Validation Checklist

Before completion verify:

- User needs validated
- User flows completed
- Wireframes reviewed
- Prototype tested
- Accessibility evaluated
- Stakeholder feedback incorporated

---

# Failure Conditions

Stop and request clarification if:

- User research is unavailable
- Business goals are unclear
- Accessibility requirements are missing
- User feedback is insufficient
- Success metrics are undefined

---

# Rules

- Design for users first.
- Validate decisions with research.
- Keep interfaces simple.
- Prioritize accessibility.
- Iterate continuously.

---

# Success Criteria

This skill succeeds when:

- user goals are achieved efficiently
- usability issues are minimized
- accessibility standards are met
- user satisfaction improves
- business objectives are supported
