# Design System

## Purpose

This skill establishes a unified design system that ensures consistency, usability, accessibility, and scalability across all products and platforms.

The design system serves as the single source of truth for UI design and front-end implementation.

---

# When to Use

Execute when:

- Designing new user interfaces
- Building reusable UI components
- Creating design libraries
- Developing front-end applications
- Improving accessibility
- Standardizing product experiences

---

# Objectives

Ensure the design system is:

- Consistent
- Reusable
- Accessible
- Scalable
- Maintainable
- User-centered

---

# Inputs

Expected inputs:

- Product Requirements
- Brand Guidelines
- UX Research
- Accessibility Requirements
- Front-end Architecture
- Component Inventory

---

# Design Principles

Follow these principles:

- Consistency
- Simplicity
- Clarity
- Accessibility
- Responsiveness
- Reusability

Every component should solve a common design problem.

---

# Design Tokens

Define reusable tokens for:

- Colors
- Typography
- Spacing
- Border Radius
- Shadows
- Elevation
- Breakpoints
- Motion

Keep tokens platform-independent where possible.

---

# Components

Maintain standardized components including:

- Buttons
- Forms
- Inputs
- Tables
- Navigation
- Cards
- Dialogs
- Alerts
- Icons
- Layouts

Each component should include usage guidelines and examples.

---

# Accessibility

Comply with:

- WCAG guidelines
- Keyboard Navigation
- Screen Reader Support
- Color Contrast Requirements
- Focus Indicators
- Semantic HTML

Accessibility is a core requirement, not an optional enhancement.

---

# Responsive Design

Support:

- Mobile
- Tablet
- Desktop
- Large Displays

Use responsive layouts and flexible design patterns.

---

# Documentation

Each component should document:

- Purpose
- Properties
- Variants
- States
- Usage Examples
- Accessibility Notes
- Design Tokens Used

Keep documentation synchronized with implementation.

---

# Governance

Define:

- Contribution Process
- Review Process
- Versioning Strategy
- Deprecation Policy
- Change Approval

Maintain consistency across all teams.

---

# Validation Checklist

Before completion verify:

- Components standardized
- Tokens documented
- Accessibility verified
- Responsive behavior tested
- Documentation updated
- Governance process defined

---

# Failure Conditions

Stop and request clarification if:

- Brand guidelines are unavailable
- Accessibility requirements are undefined
- Component scope is unclear
- Documentation is incomplete
- Governance is not established

---

# Outputs

Generate:

- Design System Guide
- Component Library Specification
- Design Token Catalog
- Accessibility Checklist
- Contribution Guide

---

# Rules

- Reuse components before creating new ones.
- Keep design and implementation synchronized.
- Prioritize accessibility by default.
- Version all design system changes.
- Maintain consistency across products.

---

# Success Criteria

This skill succeeds when:

- user interfaces are visually consistent
- components are reusable across projects
- accessibility standards are met
- development and design stay aligned
- the design system scales with product growth
