# Documentation

## Purpose

This skill establishes documentation standards to ensure knowledge is accurate, discoverable, maintainable, and useful throughout the software lifecycle.

Documentation should support developers, operators, stakeholders, and end users.

---

# When to Use

Execute when:

- Starting a new project
- Designing system architecture
- Developing APIs
- Writing user guides
- Creating operational manuals
- Recording technical decisions

---

# Objectives

Ensure documentation is:

- Clear
- Accurate
- Consistent
- Versioned
- Searchable
- Maintainable

---

# Inputs

Expected inputs:

- Business Requirements
- Product Requirements
- Architecture Documents
- API Specifications
- Source Code
- Operational Procedures

---

# Documentation Types

Maintain documentation for:

- Product Documentation
- Technical Documentation
- API Documentation
- Architecture Documentation
- Operational Runbooks
- User Guides
- Developer Guides
- Release Notes
- Changelogs

---

# Documentation Structure

Each document should include:

- Title
- Purpose
- Scope
- Audience
- Prerequisites
- Main Content
- Examples
- References
- Version History

Use a consistent structure across all documents.

---

# Writing Guidelines

Ensure content is:

- Concise
- Unambiguous
- Well-organized
- Technically accurate
- Free of unnecessary jargon

Write for the intended audience.

---

# Maintenance

Review documentation:

- After major releases
- After architectural changes
- After incident resolution
- During retrospectives
- On a scheduled basis

Remove obsolete information promptly.

---

# Version Control

Track:

- Document Version
- Author
- Review Date
- Change Summary
- Approval Status

Store documentation in version control whenever possible.

---

# Validation Checklist

Before completion verify:

- Purpose defined
- Audience identified
- Content reviewed
- Examples included
- References updated
- Version history maintained

---

# Failure Conditions

Stop and request clarification if:

- Purpose is unclear
- Information is incomplete
- References are missing
- Content conflicts with implementation
- Ownership is undefined

---

# Outputs

Generate:

- Documentation Guide
- Documentation Template
- Review Checklist
- Maintenance Plan

---

# Rules

- Keep documentation close to the code.
- Update documentation with every significant change.
- Prefer examples over lengthy explanations.
- Remove outdated content.
- Review documentation regularly.

---

# Success Criteria

This skill succeeds when:

- documentation is accurate
- knowledge is easy to find
- teams can onboard efficiently
- documentation stays synchronized with implementation
- stakeholders trust the documentation
