# Postmortem

## Purpose

This skill creates comprehensive, blameless postmortem reports after significant incidents, outages, or project failures.

The objective is to learn from failures, improve systems and processes, and prevent similar incidents in the future.

---

# When to Use

Execute when:

- Major production incidents occur
- Critical outages are resolved
- High-impact defects are fixed
- Security incidents are contained
- Significant project failures require analysis

---

# Objectives

Produce postmortems that are:

- Blameless
- Evidence-based
- Actionable
- Transparent
- Improvement-focused

---

# Inputs

Expected inputs:

- Incident Report
- Timeline
- System Logs
- Monitoring Data
- Root Cause Analysis
- Stakeholder Feedback

---

# Postmortem Structure

Generate the following sections:

- Executive Summary
- Incident Overview
- Business Impact
- Timeline
- Root Cause Analysis
- Contributing Factors
- What Went Well
- What Went Wrong
- Lessons Learned
- Corrective Actions
- Preventive Actions
- Owners
- Due Dates
- Success Metrics
- References

---

# Timeline

Document:

- Detection
- Escalation
- Investigation
- Mitigation
- Resolution
- Recovery

Use chronological order with timestamps whenever available.

---

# Root Cause Analysis

Identify:

- Primary Cause
- Contributing Factors
- Process Gaps
- Technical Weaknesses
- Organizational Factors

Support findings with evidence.

---

# Lessons Learned

Capture:

- Successful Practices
- Failed Assumptions
- Communication Improvements
- Technical Improvements
- Operational Improvements

Focus on organizational learning.

---

# Action Items

For each action include:

- Description
- Owner
- Priority
- Due Date
- Success Criteria
- Status

Track actions until completion.

---

# Validation Checklist

Verify:

- Timeline complete
- Root cause validated
- Lessons documented
- Actions assigned
- Owners identified
- Follow-up scheduled

---

# Failure Conditions

Stop if:

- Root cause is unverified
- Timeline is incomplete
- Evidence is insufficient
- Action items are missing
- Stakeholders have not reviewed

---

# Outputs

Generate:

- Postmortem Report
- Lessons Learned
- Improvement Plan
- Action Register

---

# Rules

- Never assign personal blame.
- Focus on systems and processes.
- Support conclusions with evidence.
- Define measurable improvements.
- Track actions to completion.

---

# Success Criteria

This skill succeeds when:

- root causes are understood
- organizational learning is captured
- preventive actions are implemented
- similar incidents become less likely
