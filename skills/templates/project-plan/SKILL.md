# Project Plan

## Purpose

This skill creates a comprehensive Project Plan that defines how a project will be executed, monitored, and successfully delivered.

The project plan aligns stakeholders around scope, schedule, resources, risks, and measurable outcomes.

---

# When to Use

Execute when:

- Starting a new project
- Planning a major initiative
- Defining project scope
- Preparing delivery schedules
- Coordinating cross-functional teams
- Managing project execution

---

# Objectives

Produce a Project Plan that is:

- Clear
- Realistic
- Measurable
- Traceable
- Executable

---

# Inputs

Expected inputs:

- Business Requirements
- Product Requirements
- Stakeholder Expectations
- Budget Constraints
- Resource Availability
- Timeline Requirements

---

# Project Plan Structure

Generate the following sections:

- Executive Summary
- Project Overview
- Objectives
- Scope
- Out of Scope
- Deliverables
- Milestones
- Timeline
- Work Breakdown Structure (WBS)
- Roles & Responsibilities
- Resource Plan
- Risk Management
- Communication Plan
- Quality Plan
- Success Metrics
- Approval

---

# Scope

Define:

- In Scope
- Out of Scope
- Assumptions
- Constraints
- Dependencies

Clearly establish project boundaries.

---

# Schedule

Document:

- Project Start Date
- Major Milestones
- Sprint or Phase Plan
- Delivery Dates
- Critical Path

Include dependencies between activities.

---

# Resources

Identify:

- Team Members
- Roles
- Skills
- Budget
- Tools
- External Dependencies

Allocate resources based on project priorities.

---

# Risk Management

Document:

- Project Risks
- Technical Risks
- Business Risks
- Schedule Risks
- Mitigation Strategies
- Contingency Plans

Review risks regularly throughout execution.

---

# Communication Plan

Define:

- Stakeholders
- Reporting Frequency
- Meeting Cadence
- Status Reports
- Escalation Process

Ensure transparent communication.

---

# Success Metrics

Measure:

- Schedule Adherence
- Budget Performance
- Scope Completion
- Quality Targets
- Customer Satisfaction
- Business Outcomes

Use measurable KPIs.

---

# Validation Checklist

Verify:

- Objectives defined
- Scope approved
- Timeline realistic
- Risks documented
- Resources allocated
- Stakeholders aligned

---

# Failure Conditions

Stop if:

- Objectives are unclear
- Scope is incomplete
- Resources are unavailable
- Timeline is unrealistic
- Risks are not assessed

---

# Outputs

Generate:

- Project Plan
- Milestone Schedule
- Risk Register
- Resource Plan
- Communication Plan

---

# Rules

- Align the plan with business objectives.
- Define measurable milestones.
- Keep scope under control.
- Assign clear ownership.
- Review and update the plan regularly.

---

# Success Criteria

This skill succeeds when:

- stakeholders approve the plan
- execution can begin without ambiguity
- risks are proactively managed
- project goals are measurable and achievable
