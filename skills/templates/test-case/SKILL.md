# Test Case

## Purpose

This skill creates comprehensive Test Cases that verify software behavior against requirements and acceptance criteria.

Each test case should be clear, repeatable, and traceable to business and technical requirements.

---

# When to Use

Execute when:

- Validating new features
- Preparing regression testing
- Supporting release testing
- Creating automated tests
- Performing acceptance testing
- Documenting quality assurance

---

# Objectives

Produce test cases that are:

- Clear
- Repeatable
- Traceable
- Testable
- Maintainable

---

# Inputs

Expected inputs:

- Product Requirements
- User Stories
- Acceptance Criteria
- API Specifications
- UI Designs
- Business Rules

---

# Test Case Structure

Generate the following sections:

- Test Case ID
- Title
- Objective
- Related Requirement
- Priority
- Preconditions
- Test Data
- Test Steps
- Expected Result
- Actual Result
- Status
- Notes

---

# Test Information

Document:

- Feature Name
- Module
- Test Type
- Environment
- Tester
- Execution Date

Ensure each test case has a unique identifier.

---

# Test Steps

For each step include:

- Step Number
- Action
- Input Data
- Expected Outcome

Keep steps simple and reproducible.

---

# Expected Results

Specify:

- Success Criteria
- Validation Rules
- Error Messages
- UI Changes
- API Responses

Make expected outcomes measurable.

---

# Traceability

Link every test case to:

- Requirement ID
- User Story
- Acceptance Criteria
- Defect (if applicable)

Maintain complete traceability.

---

# Validation Checklist

Verify:

- Requirement linked
- Preconditions defined
- Steps complete
- Expected results measurable
- Test data available
- Traceability maintained

---

# Failure Conditions

Stop if:

- Requirement is unclear
- Acceptance criteria are missing
- Test environment is unavailable
- Test data is incomplete
- Expected results are undefined

---

# Outputs

Generate:

- Test Case
- Execution Checklist
- Traceability Matrix
- Test Data Specification

---

# Rules

- One objective per test case.
- Write reproducible steps.
- Keep expected results explicit.
- Maintain requirement traceability.
- Update test cases as requirements evolve.

---

# Success Criteria

This skill succeeds when:

- requirements are fully validated
- defects can be reproduced
- test execution is consistent
- traceability is complete
