# Test Plan

## Purpose

This skill creates a complete Test Plan that defines the testing strategy, objectives, scope, schedule, resources, and quality gates for a software project.

The Test Plan ensures all stakeholders share a common understanding of how product quality will be validated.

---

# When to Use

Execute when:

- Planning software testing
- Preparing a release
- Starting a QA phase
- Defining quality strategy
- Coordinating testing activities
- Managing project risks

---

# Objectives

Produce a Test Plan that is:

- Complete
- Traceable
- Measurable
- Risk-based
- Actionable

---

# Inputs

Expected inputs:

- Product Requirements
- User Stories
- Acceptance Criteria
- Technical Architecture
- Release Schedule
- Risk Assessment

---

# Test Plan Structure

Generate the following sections:

- Executive Summary
- Test Objectives
- Test Scope
- Out of Scope
- Test Strategy
- Test Environment
- Test Types
- Resources
- Schedule
- Entry Criteria
- Exit Criteria
- Risks
- Deliverables
- Approval

---

# Test Strategy

Define:

- Manual Testing
- Automated Testing
- Functional Testing
- Non-functional Testing
- Regression Testing
- User Acceptance Testing

Select approaches appropriate to project risk.

---

# Test Environment

Document:

- Hardware
- Software
- Operating Systems
- Browsers
- Mobile Devices
- Test Data

Ensure environments closely match production.

---

# Test Types

Include:

- Unit Testing
- Integration Testing
- System Testing
- End-to-End Testing
- Performance Testing
- Security Testing
- Accessibility Testing

---

# Entry & Exit Criteria

Entry Criteria:

- Requirements approved
- Test environment ready
- Build available
- Test data prepared

Exit Criteria:

- Critical defects resolved
- Regression passed
- Acceptance criteria satisfied
- Release approved

---

# Risk Management

Identify:

- Testing Risks
- Schedule Risks
- Resource Risks
- Technical Risks

Define mitigation strategies for each.

---

# Validation Checklist

Verify:

- Scope defined
- Strategy approved
- Environment prepared
- Risks documented
- Exit criteria measurable
- Stakeholders aligned

---

# Failure Conditions

Stop if:

- Requirements are incomplete
- Test environment is unavailable
- Test strategy is undefined
- Critical dependencies are missing
- Acceptance criteria are unclear

---

# Outputs

Generate:

- Test Plan
- Test Schedule
- Resource Plan
- Risk Register
- Test Environment Specification

---

# Rules

- Base testing on risk.
- Define measurable quality gates.
- Ensure traceability to requirements.
- Include both manual and automated testing.
- Review the plan before execution.

---

# Success Criteria

This skill succeeds when:

- testing activities are clearly planned
- quality objectives are measurable
- risks are managed proactively
- stakeholders approve the testing approach
