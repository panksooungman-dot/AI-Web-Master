# UX Review

## Purpose

This skill evaluates the designed user experience before development begins.

The objective is to identify usability issues, reduce user friction, and verify that the design supports business goals and user needs.

UX Review validates the experience—not the visual aesthetics.

---

# When to Use

Execute after:

- design/ui-design

Execute before:

- prototype
- development/frontend

---

# Objectives

Validate that the design is:

- Easy to understand
- Efficient to use
- Consistent
- Accessible
- Error tolerant
- Business aligned

---

# Inputs

Expected inputs:

- PRD
- User Stories
- Information Architecture
- User Flow
- UI Design
- Design System

---

# Review Areas

Evaluate:

- Navigation
- Information Hierarchy
- Task Completion
- Learnability
- Discoverability
- Accessibility
- Feedback
- Error Recovery

---

# User Journey Review

For every user role verify:

- Entry Point
- Navigation
- Decision Points
- Success Path
- Exit Path

Ensure every goal can be completed without unnecessary steps.

---

# Usability Checklist

Review:

- Navigation clarity
- Button placement
- Form usability
- Search experience
- Empty states
- Error messages
- Loading states
- Confirmation dialogs

---

# Accessibility Review

Verify:

- WCAG compliance
- Keyboard navigation
- Screen reader support
- Color contrast
- Focus indicators
- Touch target sizes

Accessibility issues must be documented.

---

# Consistency Review

Check consistency of:

- Layout
- Components
- Typography
- Icons
- Colors
- Terminology
- Interactions

Users should not encounter conflicting patterns.

---

# UX Problems

Document every issue using:

- Problem
- Impact
- Severity
- Recommendation

Severity Levels:

- Critical
- High
- Medium
- Low

---

# UX Workflow

```
Review User Flow

↓

Review UI Screens

↓

Evaluate User Journey

↓

Identify UX Issues

↓

Prioritize Findings

↓

Recommend Improvements

↓

Revalidate

↓

Approve

↓

Pass to Prototype
```

---

# Outputs

Generate:

- UX Review Report
- Usability Findings
- Accessibility Report
- Improvement Recommendations
- Approval Status

---

# Rules

- Review from the user's perspective.
- Focus on solving user problems.
- Base findings on evidence.
- Prioritize high-impact improvements.
- Avoid subjective opinions without justification.

---

# Validation Checklist

Before completion verify:

- Navigation reviewed
- User journeys validated
- Accessibility checked
- UX issues documented
- Recommendations prioritized
- Design approved

---

# Success Criteria

This skill succeeds when:

- major usability issues are resolved
- accessibility standards are met
- user flows are intuitive
- business goals remain supported
- design is ready for prototyping
