# Planning

> **Status: draft** — minimal structure only. Expand with concrete planning artifacts/templates
> as real cases accumulate.

## Purpose

Turns an Intake record into a scoped, sequenced plan — what will be built, in what order, and
what is explicitly out of scope. See also `skills/templates/prd/SKILL.md` and
`skills/templates/project-plan/SKILL.md` for the document formats this stage typically
produces.

## Position in Pipeline

```
Intake → [ Planning ] → Design → Development → Quality → Release → Operations
```

## Responsibilities

- Convert the Intake record into a scoped plan
- Identify which Expert Skills (`skills/experts/`) the work needs
- Flag architecturally significant decisions for Design rather than deciding them here
